# ECS 模块 — 编码约定

> 写 `Assets/Scripts/ECS/` 下任何代码时自动加载。
> **当前状态：Phase 3 — 最小骨架已接入，尚未全面迁移。**

---

## 当前文件

| 文件 | 作用 |
|------|------|
| `BattleEcsComponents.cs` | ECS 组件定义（IComponentData） |
| `BattleEcsRuntime.cs` | ECS World 生命周期管理 |

---

## 改造原则

1. **增量迁移，不推倒重来** — 先在 ECS 中跑新功能，逐步替换 Battle.cs 中的逻辑
2. **Burst/Job 先行** — 先确认算法在 Burst IJob 中能跑，再封装为 ISystem
3. **保持 Battle.cs 现有 API 不变** — 确保外部回调（UI_Battle 等）不受影响

---

## 编码规范

### 组件定义
```csharp
public struct BattleUnitComponent : IComponentData {
    public int unitId;
    public float health;
    public BattleVector2 position;
}
```

### System 定义
```csharp
[BurstCompile]
public partial struct UnitMoveSystem : ISystem {
    public void OnUpdate(ref SystemState state) {
        // Burst 兼容代码
    }
}
```

---

## 禁止事项

- ❌ 在 ECS System 中引用 `MonoBehaviour` / `GameObject`
- ❌ 在热路径 System 中使用托管对象
- ❌ 绕过 `BattleEcsRuntime` 直接创建 World
- ❌ 一次性大规模重写 — 保持增量
